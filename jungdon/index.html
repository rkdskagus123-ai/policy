정돈 개인정보 처리방침

시행일: 2026년 6월 5일

정돈은 사용자가 직접 기록한 물건, 식재료, 준비한 음식, 보관 위치, 수량, 메모, 사용자가 입력한 날짜를 기기 안에서 정리하도록 돕는 로컬 우선 앱입니다. 현재 릴리스는 계정, 로그인, 서버 동기화, 가족 공유 기능을 제공하지 않습니다.

이 문서는 현재 Google Play 제출 후보의 데이터 처리 방식을 설명합니다.

1. 처리하는 정보

정돈은 기능 제공을 위해 다음 정보를 기기 안에서 처리할 수 있습니다.

  • 사용자가 입력한 항목 이름, 종류, 수량, 단위, 메모
  • 사용자가 만든 장소, 보관함, 세부 위치
  • 사용자가 입력한 등록일과 소비기한 날짜
  • 소비기한 알림 설정
  • 사용자가 시작한 음성 찾기 문장
  • 사용자가 선택하거나 시스템 스캐너로 촬영한 영수증/사진과 그 이미지에서 추출한 후보 텍스트
  • 사용자가 직접 만든 JSON 백업 파일
  • 광고 제거 구매가 적용되었는지 판단하기 위한 기기 안의 로컬 상태

광고가 활성화된 배포판에서는 Google Mobile Ads SDK가 비개인화 광고 요청, 광고 제공, 측정, 부정 사용 방지를 위해 필요한 정보를 처리할 수 있습니다. 광고 제거 구매는 Google Play 결제를 통해 처리됩니다. 정돈은 결제 카드 번호나 결제 수단의 상세 정보를 직접 저장하지 않습니다.

2. 이용 목적

정돈은 위 정보를 다음 목적에만 사용합니다.

  • 물건, 식재료, 준비한 음식 목록 관리
  • 보관 위치 확인과 검색
  • 사용자가 입력한 날짜 기준의 소비기한 목록 표시
  • 사용자가 켠 경우 소비기한 로컬 알림 제공
  • 영수증/사진에서 후보 항목을 만들고 사용자가 확인한 뒤 등록 화면으로 이어가기
  • 바코드 값으로 로컬 저장 항목 또는 공개 제품명 후보 확인
  • 사용자가 확인한 경우 외부 브라우저에서 10000recipe 레시피 검색 열기
  • 사용자가 직접 요청한 JSON 백업 파일 생성, 미리보기, 공유, 복원
  • 광고 표시, 광고 제거 구매 처리, 광고 표시 여부 판단

영수증/사진 인식 결과는 자동으로 저장되지 않습니다. 후보는 확인 화면을 거치며, 사용자가 등록 화면에서 저장해야 항목이 만들어집니다. 인식 결과는 틀리거나 누락될 수 있으므로 저장 전에 사용자가 확인해야 합니다.

3. 외부 전송과 공유

현재 앱 코드에는 정돈 서버로 항목, 위치, 영수증 이미지, OCR 텍스트, 음성 녹음, 음성 인식 문장, 백업 파일을 업로드하는 기능이 없습니다. 계정이나 서버 동기화 기능도 제공하지 않습니다.

사용자가 제품형 식품 바코드를 스캔하면 제품명 후보 조회를 위해 바코드 값이 Open Food Facts로 전송될 수 있습니다. 이 조회에는 항목 이름, 보관 위치, 메모, 영수증 이미지, OCR 텍스트, 음성 녹음, 음성 인식 문장이 함께 전송되지 않습니다.

사용자가 레시피 검색 열기를 확인하면 재고 기반 레시피 검색어가 외부 브라우저를 통해 10000recipe로 전달될 수 있습니다. 정돈은 10000recipe 페이지를 스크랩, 파싱, 복사하거나 저장하지 않으며 정돈 자체 콘텐츠처럼 표시하지 않습니다.

광고가 활성화된 배포판에서는 Google Mobile Ads SDK가 비개인화 광고 요청, 광고 제공, 측정, 부정 사용 방지를 위해 Google 또는 광고 관련 파트너와 데이터를 주고받을 수 있습니다. 광고 제거 구매는 Google Play 결제 시스템을 통해 처리됩니다.

정돈은 개인정보를 판매하지 않습니다. 사용자가 JSON 백업 파일을 직접 공유하면 그 파일을 받은 사람이나 서비스의 처리 방식이 적용될 수 있습니다. 백업 파일에는 개인적인 항목 이름, 위치, 날짜, 메모가 포함될 수 있으므로 신뢰할 수 있는 곳에서만 다루어야 합니다.

4. 권한

정돈은 소비기한 알림을 제공하기 위해 알림 권한을 요청할 수 있습니다. 알림 권한은 사용자가 알림 기능을 켤 때 요청합니다.

정돈은 사용자가 시작하는 음성 찾기를 위해 마이크 권한을 요청할 수 있습니다. 음성 찾기는 로컬 검색 보조 기능이며, 현재 앱은 음성 파일을 정돈 서버나 클라우드 음성 서비스로 업로드하지 않습니다.

영수증/사진 입력은 시스템 스캐너나 시스템 선택 화면을 통해 진행됩니다. 현재 릴리스는 앱 자체 카메라 권한, 사진/미디어 권한, 위치 권한, 계정 권한을 요청하지 않는 것을 목표로 합니다.

광고가 활성화된 배포판에는 광고 제공을 위한 인터넷 및 네트워크 상태 권한과 광고 제거 구매를 위한 Google Play 결제 권한이 포함될 수 있습니다.

5. 보관과 삭제

확인 후 저장한 항목 데이터는 이 기기 안에 보관됩니다. 영수증/사진 입력에 사용한 앱 생성 임시 이미지 사본은 후보 처리 후 정리되며, 원본 사진을 정돈의 앱 데이터로 보관하지 않습니다. 사용자가 기존 갤러리 이미지를 선택한 경우 원본 파일은 사용자의 기기와 선택한 저장 위치의 관리 방식에 따릅니다.

사용자가 앱에서 항목을 삭제하면 해당 항목은 활성 목록에서 제외됩니다. 앱을 삭제하면 OS 백업과 기기 설정에 따라 앱 데이터가 제거될 수 있습니다. 사용자가 직접 만든 JSON 백업 파일은 사용자가 해당 파일을 삭제하거나 파일을 보관한 위치에서 제거하기 전까지 남아 있을 수 있습니다.

6. 외부 서비스와 SDK

현재 Android 빌드에는 바코드 스캔, 문서 스캔, 텍스트 인식에 필요한 Google Play services 및 ML Kit 관련 구성요소가 포함될 수 있습니다. 정돈은 이 구성요소를 앱 기능 제공 목적 안에서 사용합니다.

바코드 제품명 후보 조회에는 Open Food Facts의 공개 제품 데이터가 사용될 수 있습니다. Open Food Facts 데이터는 ODbL 라이선스 조건과 출처 표시 요구를 따릅니다.

10000recipe 레시피 검색은 사용자가 확인한 경우 외부 브라우저에서 열립니다. 정돈은 외부 레시피 페이지의 내용을 저장하지 않으며 자체 콘텐츠로 제공하지 않습니다.

광고가 활성화된 Android 배포판에는 Google Mobile Ads SDK가 포함될 수 있습니다. 광고 제거 구매에는 Google Play 결제 시스템과 관련 라이브러리가 사용됩니다. Play Console SDK 안내나 검토 결과가 추가 확인을 요구하는 경우, 최종 제출 전 이 처리방침과 Data safety 답변을 다시 확인해야 합니다.

7. 아동

정돈은 현재 어린이를 주 대상으로 설계된 앱이 아닙니다. 대상 연령이나 아동 관련 제품 방향이 바뀌는 경우 Google Play 대상 연령 및 아동 관련 정책을 다시 확인해야 합니다.

8. 음식 안전과 인식 정확도

정돈은 사용자가 입력한 날짜를 정리해 보여주고 알림을 제공하는 도구입니다. 음식의 안전 여부를 판단하거나 보장하지 않습니다.

영수증/사진에서 추출한 후보 항목은 정확하지 않을 수 있으며 사용자가 저장 전에 내용을 확인해야 합니다. 바코드 제품명 후보도 틀리거나 없을 수 있고, 수량과 가격은 자동으로 판단하지 않습니다.

외부 브라우저에서 열린 10000recipe 검색 결과와 레시피 내용은 10000recipe가 제공하는 외부 콘텐츠입니다. 정돈은 해당 내용을 저장하지 않으며 자체 콘텐츠로 제공하지 않습니다.

이 처리방침이 변경되면 문서의 시행일을 갱신합니다. 계정, 서버 동기화, 분석, 클라우드 OCR, 클라우드 백업, 개인 맞춤 광고, 대상 지역, 광고 또는 결제 처리 방식이 바뀌는 경우 해당 변경을 배포하기 전에 처리방침과 Google Play Data safety 답변을 다시 검토해야 합니다.
